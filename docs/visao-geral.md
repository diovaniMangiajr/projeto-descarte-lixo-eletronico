## 2. Identificação e Contextualização do Problema
O avanço tecnológico acelerado e o consumismo resultam em uma troca constante de dispositivos eletrônicos. 
Consequentemente, a geração de resíduos eletrônicos (e-waste) cresce exponencialmente. 
O problema central abordado por este projeto é a **falta de informação e de infraestrutura acessível para o descarte correto desses materiais** 
por parte da população. O descarte irregular em lixo comum causa graves impactos ambientais e riscos à saúde pública devido à presença de metais 
pesados e substâncias tóxicas.

## 3. Justificativa da Proposta de Solução
Para mitigar os impactos ambientais e promover a conscientização, é fundamental conectar os cidadãos que desejam descartar seus eletrônicos aos pontos de coleta adequados. A solução proposta justifica-se pela necessidade de centralizar, organizar e disponibilizar essas informações de forma clara e interativa, facilitando o acesso da população a locais de descarte seguro e permitindo que administradores realizem a gestão eficiente desses dados.

## 4. Visão do Produto e Objetivos
**Visão do Produto:** Uma plataforma web confiável e de fácil utilização que mapeia pontos de coleta de lixo eletrônico, incentivando a sustentabilidade e o descarte ecologicamente correto.

**Objetivos do Projeto:**
* Desenvolver uma aplicação web responsiva para visualização e busca de pontos de coleta.
* Criar um painel administrativo seguro para a gestão (CRUD) dos pontos de coleta.
* Aplicar, na prática, os conceitos de Engenharia de Software, incluindo modelagem, arquitetura, padrões de projeto e testes.

## 5. Escopo Inicial da Aplicação Web (MVP)
O escopo inicial (Mínimo Produto Viável) foca nas funcionalidades essenciais para viabilizar a plataforma:
1. **Área Pública (Cidadão):**
   * Interface web com listagem de pontos de coleta de lixo eletrônico.
2. **Área Administrativa (Gestão):**
   * Sistema de autenticação (login/logout) seguro para administradores.
   * Dashboard para gestão (Cadastro, Leitura, Atualização e Exclusão - CRUD) dos pontos de coleta (incluindo nome, endereço e tipos de resíduos aceitos).

## 6. Stack Tecnológico Selecionado
Para garantir uma arquitetura robusta, moderna e adequada à proposta de aplicação web, o grupo definiu a seguinte stack:
* **Frontend:** React
* **Backend:** Java com Spring Boot
* **Banco de Dados:** PostgreSQL
* **Infraestrutura/Ambiente:** GitHub (para versionamento e gestão do projeto).

## 7. Organização Inicial do Projeto (Metodologia Ágil)
O desenvolvimento seguirá as diretrizes do **Scrum adaptado ao contexto acadêmico**, garantindo entregas incrementais ao longo de 8 sprints:
* **Product Backlog:** Centralizado no GitHub Projects, gerido pelo Product Owner, contendo as User Stories priorizadas.
* **Sprints:** Ciclos de desenvolvimento iterativo com definição clara do Sprint Backlog (tarefas selecionadas para a iteração).
* **Rastreabilidade e Versionamento:** Uso contínuo de Issues, Labels e Milestones vinculados ao GitHub, garantindo o registro de evolução sob a supervisão do Scrum Master.
